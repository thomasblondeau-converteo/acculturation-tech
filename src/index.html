La Donnée : le carburant de tout

Avant de parler IA ou pipeline, comprendre ce qu'est vraiment une donnée — et pourquoi seule, elle ne vaut rien.

Problème Business
"Nos données clients existent depuis 5 ans — mais on ne sait toujours pas quel canal génère nos meilleurs clients."

La donnée brute ne vaut rien seule. Sa valeur naît de la chaîne complète : collecte → stockage → traitement → analyse → action.

SQL vs NoSQL

SQL · Relationnel
Structure rigide & fiable

Tables liées, schéma fixe, garanties ACID. Standard pour les données transactionnelles.

SELECT user, revenue FROM orders
WHERE date > '2024-01-01'
Ex : PostgreSQL, MySQL, BigQuery
NoSQL · Flexible
Flexibilité & scalabilité

Pas de schéma imposé. Idéal pour des données hétérogènes à grande échelle ou en temps réel.

db.users.find({
  "plan": "premium",
  "city": "Paris"
})
Ex : MongoDB, Firestore, DynamoDB

💡 L'API est un contrat : elle définit exactement quelles données peuvent être demandées, dans quel format, avec quelles permissions.

GET vs POST : les deux gestes fondamentaux

GETLire une ressource

Je veux obtenir de l'information. Idempotente : peut être appelée N fois sans rien modifier côté serveur.

GET /api/v1/clients?id=123
→ {"name": "Fnac Darty", "revenue": 4200000}
POSTCréer / Envoyer des données

Je veux envoyer de l'information. Les données sont dans le corps de la requête.

POST /api/v1/leads
Body: {"email": "contact@fnac.fr"}
→ 201 Created

✓ API temps réel (moderne)
✓Données synchronisées en continu
✓Toujours à jour (latence < 1 seconde)
✓Logs et monitoring intégrés

L'écosystème IA : Modèle, LLM & Agent

Trois concepts distincts que tout le monde confond. Les comprendre, c'est parler le même langage que les équipes tech.

📚
Données d'entraînement
Des millions d'exemples labellisés
→ ajustement →

Modèle entraîné

Les 3 types d'apprentissage machine

🏷
Supervisé
Avec exemples connus
Le modèle apprend à partir de données déjà labellisées par des humains.
Cas client : prédiction de churn, scoring de leads
🔍
Non supervisé
Auto-découverte
Le modèle découvre lui-même des structures sans labels prédéfinis.
Cas client : segmentation client, détection d'anomalies
🎮

Essai / erreur / récompense
L'agent maximise une récompense en testant des actions dans un environnement.

Le Deep Learning — ce qui rend les LLM possibles
Les LLM sont des réseaux de neurones profonds (milliards de paramètres) entraînés sur des centaines de milliards de mots. L'architecture Transformer (Google, 2017) est la clé : elle permet au modèle de comprendre le sens des mots selon leur contexte, pas juste leur forme.
~1T
paramètres GPT-4

Modèle · LLM · Agent : les vraies différences

⚙️
Modèle ML
Spécialisé, borné

Prédit une chose précise. Entrée → Calcul → Sortie.

Ex : prédit si un email est spam
🧠
LLM
Général, conversationnel

Comprend et génère du texte. Répond à des instructions en langage naturel.

Ex : GPT-4.1 / o3, Claude Sonnet 4.6, Gemini 2.5 Pro, Mistral Large 2
🤖
Agent IA
Autonome, actif

LLM + outils + mémoire + planification. Prend des décisions en plusieurs étapes.

Ex : agent analyse verbatims + envoie rapport

Concepts clés : Tokens & Fenêtre de contexte

Comprendre ces 2 notions, c'est comprendre les limites et les coûts des LLM.

🔤
Token

Unité de texte traitée par le LLM. Ni mot, ni lettre : ≈ 0,75 mot en moyenne. "converteo" = 3 tokens.

💶 Les API facturent au nombre de tokens (entrée + sortie)
📐
Contexte

Nombre max de tokens que le modèle peut "voir" en même temps. Détermine ce qu'on peut lui soumettre en une requête.

📄 128k tokens ≈ 100 pages PDF
🗺️
Embedding

Représentation mathématique du sens d'un texte sous forme de vecteur. Clé pour la recherche sémantique et le RAG.

✅ "Réponds au format : 1) Insight principal, 2) Chiffres clés, 3) Recommandation action"

Chain of Thought — Faites raisonner le modèle

Demander au LLM de penser étape par étape améliore drastiquement la qualité sur les tâches complexes. Ne donnez pas directement la réponse attendue : guidez le raisonnement.

✅ "Analyse d'abord les données brutes, identifie les tendances, puis formule 3 hypothèses, et enfin donne ta recommandation finale."

CRITIQUE

Un LLM peut inventer des faits avec confiance : chiffres, citations, noms, dates, références — tout en paraissant absolument certain. Ce n'est pas un bug, c'est une limite structurelle du modèle.

⚠️ Limites et risques de l'IA agentique

💸
Coût d'exécution
Un agent qui itère 20 fois peut générer des coûts API importants. Toujours fixer un budget max par tâche.
🛡 Garde-fou : limiter le nombre de loops, timeout, budget token
🔓
Sécurité
Un agent avec accès BDD peut exécuter des requêtes dangereuses si mal prompt-injectioné.
🛡 Garde-fou : permissions strictes, validation humaine pour actions critiques
🎭
Hallucinations
Le LLM peut inventer des données ou actions qui n'existent pas.
🛡 Garde-fou : toujours sourcer les faits, validation humaine finale
♾️
Boucles infinies
Un agent mal calibré peut boucler indéfiniment sans converger.
🛡 Garde-fou : max iterations (ex: 10), timeout global, surveillance logs
✅ Règle d'or : Commencez par un humain dans la boucle (human-in-the-loop) pour les actions critiques. L'autonomie totale se gagne progressivement avec la confiance.

✓ Avec RAG : Le système récupère les bons documents, les injecte dans le contexte, et répond avec précision et sources.

🚀 RAG avancé — Techniques pour aller plus loin

🎯
Hybrid Search
Combiner recherche sémantique (vecteurs) + recherche par mots-clés (BM25)
→ Termes techniques précis (SKU, codes) + recherche sémantique
🔄
Re-ranking
Après retrieval, re-classer les top-20 chunks pour ne garder que les 5 meilleurs
→ Améliorer la précision quand beaucoup de documents similaires
🧩
Parent-Child Chunks
Indexer des petits chunks mais retourner le paragraphe/section parent pour plus de contexte
→ Éviter de perdre le contexte en découpant trop finement
🔍
Query Expansion
Reformuler la question utilisateur en plusieurs variantes avant retrieval
→ Questions vagues ou avec plusieurs interprétations possibles
🏷️
Metadata Filtering
Filtrer par date, auteur, catégorie avant la recherche vectorielle
→ Corpus multi-tenant, multi-période, multi-catégorie
📊
Contexte adaptatif
Ajuster dynamiquement le nombre de chunks selon la complexité de la question
→ Question simple : 3 chunks, question complexe : 10 chunks
