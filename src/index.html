Module 01 — Fondations

La Donnée : le carburant de tout

Avant de parler IA ou pipeline, comprendre ce qu'est vraiment une donnée — et pourquoi seule, elle ne vaut rien.

Problème Business
"Nos données clients existent depuis 5 ans — mais on ne sait toujours pas quel canal génère nos meilleurs clients."

La donnée brute ne vaut rien seule. Sa valeur naît de la chaîne complète : collecte → stockage → traitement → analyse → action.

Les 3 types de données

⊞ Structurée
⊟ Semi-structurée
⊘ Non structurée
Structurée

Format tabulaire, schéma fixe

Colonnes prédéfinies, types déclarés, requêtes SQL directes. Garanties d'intégrité strictes.

Excel / CSVPostgreSQLClics pubTransactions
# Exemple SQL

SELECT user_id, email, revenue
FROM orders
WHERE date > '2024-01-01'
Semi-structurée

Format flexible mais organisé

Structure présente mais variable. Clé/valeur, hiérarchique. Idéal pour les échanges entre systèmes.

JSONXMLEmailsLogs apps
# Exemple JSON API

{
  "user": "j.dupont",
  "event": "add_to_cart",
  "ts": 1718208000
}
Non structurée

80 % des données mondiales

Aucun format imposé. Impossible à requêter en SQL. Nécessite l'IA pour être exploitée.

ImagesVidéosAvis clientsContrats PDF
Exemples terrain
🖼 Photos produits e-commerce (Fnac)
💬 Verbatims avis clients (Trustpilot)
🎙 Enregistrements appels support

Module 02 — Architectures

Stockage & Architectures

D'où viennent les données et où vivent-elles ? Comprendre les différences entre les grandes familles de stockage.

Contexte Client — Fnac Darty
"Nos données sont partout : ERP, e-commerce, mobile, SAV... impossible de les croiser."

Le défi n'est pas de collecter plus — c'est de centraliser intelligemment.

SQL vs NoSQL

SQL · Relationnel
Structure rigide & fiable

Tables liées, schéma fixe, garanties ACID. Standard pour les données transactionnelles.

SELECT user, revenue FROM orders
WHERE date > '2024-01-01'
Ex : PostgreSQL, MySQL, BigQuery
NoSQL · Flexible
Flexibilité & scalabilité

Pas de schéma imposé. Idéal pour des données hétérogènes à grande échelle ou en temps réel.

db.users.find({
  "plan": "premium",
  "city": "Paris"
})
Ex : MongoDB, Firestore, DynamoDB

Data Lake · Data Warehouse · CDP

Data Lake
Stockage brut & flexible
●Tout y entre, dans tous les formats. Aucun schéma imposé à l'ingestion.
●Très économique, scalable à l'infini.
●Point de départ de la plupart des architectures modernes.
AWS S3Google GCSAzure Blob
Data Warehouse
Entrepôt analytique structuré
●Données nettoyées, modélisées, optimisées pour le reporting.
●Performances élevées pour les requêtes analytiques complexes.
●Gouvernance et documentation du schéma.
SnowflakeBigQueryRedshift
CDP
Customer Data Platform
●Unification des profils clients cross-canal.
●Vue 360° client, segmentation temps réel.
●Connexion directe aux outils marketing.
SegmentWeavennmParticle

APIs : les messagers entre systèmes

Comment les applications communiquent-elles entre elles ? Et pourquoi c'est fondamental pour comprendre toute architecture data.

Comment fonctionne une API ?

Point d'entrée standardisé
200 OK←Réponse JSON
🗄
Serveur / BDD
ex. PostgreSQL
💡 L'API est un contrat : elle définit exactement quelles données peuvent être demandées, dans quel format, avec quelles permissions.

GET vs POST : les deux gestes fondamentaux

GETLire une ressource

Je veux obtenir de l'information. Idempotente : peut être appelée N fois sans rien modifier côté serveur.

GET /api/v1/clients?id=123
→ {"name": "Fnac Darty", "revenue": 4200000}
POSTCréer / Envoyer des données

Je veux envoyer de l'information. Les données sont dans le corps de la requête.

POST /api/v1/leads
Body: {"email": "contact@fnac.fr"}
→ 201 Created

Partage temps réel vs échange de fichiers

❌ Échange de fichiers (legacy)
✗Export CSV manuel chaque lundi matin
✗Données déjà obsolètes à l'arrivée
✗Risques d'erreurs de version et de perte
✓ API temps réel (moderne)
✓Données synchronisées en continu
✓Toujours à jour (latence < 1 seconde)
✓Logs et monitoring intégrés

Module 05 — Intelligence Artificielle

L'écosystème IA : Modèle, LLM & Agent

Trois concepts distincts que tout le monde confond. Les comprendre, c'est parler le même langage que les équipes tech.

Comment un modèle IA apprend-il ?

📚
Données d'entraînement
Des millions d'exemples labellisés
→ ajustement →
🧠
Modèle entraîné
Capable d'inférence sur de nouvelles données

Les 3 types d'apprentissage machine

🏷
Supervisé
Avec exemples connus
Le modèle apprend à partir de données déjà labellisées par des humains.
Cas client : prédiction de churn, scoring de leads
🔍
Non supervisé
Auto-découverte
Le modèle découvre lui-même des structures sans labels prédéfinis.
Cas client : segmentation client, détection d'anomalies
🎮
Renforcement
Essai / erreur / récompense
L'agent maximise une récompense en testant des actions dans un environnement.
Cas client : optimisation d'enchères, agents autonomes

Modèle · LLM · Agent : les vraies différences

⚙️
Modèle ML
Spécialisé, borné

Prédit une chose précise. Entrée → Calcul → Sortie.

Ex : prédit si un email est spam
🧠
LLM
Général, conversationnel

Comprend et génère du texte. Répond à des instructions en langage naturel.

Ex : GPT-4, Claude, Gemini, Mistral
🤖
Agent IA
Autonome, actif

LLM + outils + mémoire + planification. Prend des décisions en plusieurs étapes.

Ex : agent analyse verbatims + envoie rapport

RAG & MCP : l'IA qui sait ce qu'elle ne sait pas

Comment connecter un LLM à vos propres données — et comment les agents communiquent avec les outils du monde réel.

RAG Retrieval-Augmented Generation

Un LLM connaît le monde jusqu'à sa date d'entraînement. Avec le RAG, on lui donne accès à vos documents en temps réel.

❌ Sans RAG : "Quels sont les chiffres Fnac Darty Q3 ?" → Je n'ai pas accès à ces données.
✓ Avec RAG : Le système récupère les bons documents, les injecte dans le contexte, et répond avec précision et sources.
1
Question posée
"Quel est le taux de churn Q3 ?"
2
Retrieval
Recherche vectorielle dans la base documentaire interne
3
Augmentation
Les extraits pertinents sont injectés dans le prompt du LLM
4
Génération
Le LLM génère une réponse ancrée sur vos données, avec sources

MCP Model Context Protocol

Standard ouvert (Anthropic, 2024) permettant à un agent IA de se connecter à n'importe quel outil externe. L'USB-C de l'IA.
